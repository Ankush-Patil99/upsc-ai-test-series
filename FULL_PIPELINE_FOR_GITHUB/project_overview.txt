## 1. Project Goal
To automatically generate strictly verified, high-quality, zero-hallucination Multiple Choice Questions (MCQs) for UPSC Civil Services aspirants. The engine uses a robust 6-Node LangGraph architecture with specialized AI agents, multi-modal Retrieval Augmented Generation (RAG) using local ChromaDB, and strict QA gating.

**Critical Design Guarantee:** For `current_affairs` type questions, the LLM is given ONLY the content retrieved from 7 scraped sources via ChromaDB. It cannot generate facts from pre-training weights — eliminating hallucination risk for recent/sensitive UPSC facts.

## 3. Architecture & The 6 Nodes
The system uses **LangGraph** to pass a shared `QuestionState` across 6 specialized LLM nodes. State management includes `SqliteSaver` for crash recovery and checkpointing.

1. **Node 1: Input Controller** (Mistral-7B)
   - Validates user input (Paper, Topic, Difficulty).
   - Expands broad topics into specific testable subtopics.
   - Checks against the syllabus mapping and ensures coverage limits.

2. **Node 2: Content Researcher** (Qwen-72B) — **CA-AWARE**
   - For `current_affairs` type: retrieves top-7 CA chunks with `source_type=CA`
     from ChromaDB (scraped real-world content). Uses a strict ANTI-HALLUCINATION
     system prompt — LLM forbidden from using parametric knowledge beyond retrieved context.
   - For other types: retrieves top-5 CA + top-3 textbook chunks normally.
   - Looks up the Static Knowledge Base JSON for constitutional facts.
   - Synthesizes a dense 300-500 word factual brief with strict source citations.

3. **Node 3: Question Drafter** (Qwen-72B)
   - Uses the research brief as its only source of truth.
   - Drafts the Question Stem, Correct Answer, and Detailed Explanation.
   - Given 5 different few-shot examples (Statement Based, Assertion-Reasoning, Match the Following, etc.) to mimic the tough UPSC phrasing.

4. **Node 4: Distractor Engineer** (Mistral-7B)
   - Generates exactly 3 highly plausible but wrong options (distractors).
   - Employs UPSC trick patterns: partial truths, swapped entities, and reversed logic.

5. **Node 5: Structure Formatter** (Mistral-7B)
   - Condenses the full explanation into a 2-3 sentence version for the website.
   - Randomly shuffles options into A, B, C, D labels.
   - Computes Difficulty Score (0.0-1.0) and assigns an estimated solving time.
   - Formats the exact JSON object schema needed for the production database.

6. **Node 6: QA Orchestrator** (Qwen-72B)
   - The Gatekeeper. It scores the output against an 8-dimension weighted rubric (Accuracy 25%, Clarity 20%, etc.).
   - Checks ChromaDB `dedup_collection` for duplicates (Cosine Similarity > 0.85 = reject).
   - **Conditional Routing:** If score < 0.75, it finds the worst-performing dimension and strictly routes back to the responsible node (max 3 retries).
   - On approval, saves the question to ChromaDB and increments coverage tracking in local SQLite.

## 4. Current Affairs (CA) Sub-System

### 4A. 7 Scraped Sources (ca_scraper.py)

All 7 sources are scraped by APScheduler every 24 hours. Content is chunked (600 chars, 80 overlap), embedded via `BAAI/bge-large-en-v1.5`, and upserted into ChromaDB RAG with `source_type=CA`.

| #  | Source                        | Method         | URL / Details                                          | GS Paper      |
|----|-------------------------------|----------------|--------------------------------------------------------|---------------|
| 1  | VisionIAS                     | PDF Download   | Monthly CA Magazine (most popular IAS prep source)     | GS1-GS4       |
| 2  | InsightsIAS                   | PDF Download   | Monthly CA Compilation                                 | GS1-GS3       |
| 3  | ForumIAS 9PM Brief            | HTML (BS4)     | Daily 9PM brief articles                               | GS1-GS3       |
| 4  | The Hindu Editorials           | RSS Feed       | Top 3 latest editorials (bypasses paywall)             | GS2,GS3,GS4   |
| 5  | Indian Express Explained       | RSS Feed       | Top 3 Explained articles (bypasses paywall)            | GS2,GS3       |
| 6  | PIB Press Releases             | HTML (BS4)     | Official GoI press releases                            | GS2,GS3       |
| 7  | Drishti IAS Environment        | HTML (BS4)     | Environment & Biodiversity articles                    | GS3           |

**Deduplication:** Each chunk is fingerprinted via SHA-256. ChromaDB's `upsert` by stable
ID makes re-runs fully idempotent — no duplicate content is ever stored.

### 4B. CA Scraper Commands
```bash
# Run all 7 sources now (one-shot):
python ca_scraper.py --run-now

# Run only a specific source:
python ca_scraper.py --run-now --source thehindu

# Start the 24h APScheduler daemon:
python ca_scraper.py --daemon

# Check CA chunk count in ChromaDB:
python ca_scraper.py --status
```

## 5. Data Ingestion & Storage

All data infrastructure is serverless and cluster-friendly to support HPC environments.

### A. RAG Database (`ChromaDB - upsc_knowledge`)
- **Textbook chunks** (`source_type=textbook`): Books, NCERTs from `data/books/`
- **CA chunks** (`source_type=CA`): Live-scraped content from 7 sources via `ca_scraper.py`
- **Embedding:** `BAAI/bge-large-en-v1.5` (local, offline)

### B. Duplicate Detection Database (`ChromaDB - upsc_questions`)
- **What it holds:** 20 Years of official UPSC PYQs + 80 Mock Test papers + all previously approved questions.
- **Embedding:** `BAAI/bge-small-en`

### C. Logging & Tracking (`SQLite - upscdb.sqlite3`)
- **`qa_logs`**: Logs QA dimensions, success margins, and pipeline performance.
- **`topic_coverage`**: Ensures balanced test bank coverage.

### D. Static Knowledge Base (`static_kb/`)
- **`syllabus_map.json`**: Built from PDFs in `data/syllabus/`.
- **`upsc_facts.json`**: Key constitutional articles, dates, and bedrock rules.

## 6. Model Infrastructure

The pipeline runs on a multi-GPU HPC environment (**2x 80GB Nvidia A100 GPUs**) using `vLLM`.

**Heavy Endpoint (Port 8001 | GPU 0+1 via TP=2):**
- **Model:** `Qwen/Qwen2.5-72B-Instruct-AWQ` (~40GB VRAM total)
- **Role:** Node 2 (Researching), Node 3 (Drafting), Node 6 (Scoring).

**Light Endpoint (Port 8002 | GPU 1, 85% utilization):**
- **Model:** `mistralai/Mistral-7B-Instruct-v0.3` (~15GB VRAM)
- **Role:** Node 1 (Routing), Node 4 (Distractors), Node 5 (Formatting).

## 9. Zero-Hallucination Architecture for Current Affairs

The anti-hallucination guarantee works in three layers:

```
Layer 1: Source Control
  ca_scraper.py fetches ONLY verified sources (VisionIAS, PIB, The Hindu, etc.)
  → Content is stored with full source metadata in ChromaDB

Layer 2: Retrieval Grounding
  node_2_researcher.py retrieves the top-7 semantically closest CA chunks
  → The LLM prompt contains ONLY this text; no external data is passed

Layer 3: Strict LLM Prompting
  RESEARCHER_SYSTEM_PROMPT_CA instructs Qwen-72B:
  "USE ONLY the retrieved CA context provided below.
   Do NOT use any knowledge from your pre-training weights."
  → Every fact must be cited with [CA:N] or it's rejected
```

This three-layer design ensures that even if a model "knows" a fact from pre-training,
it will only output it if that fact also appears in the retrieved CA context — making
the output verifiable and traceable back to a specific scraped source document.
