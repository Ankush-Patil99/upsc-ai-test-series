  3-node LangGraph pipeline:
    Node 1 — retrieve_context : Semantic search PGVector → gets top K CA chunks
    Node 2 — draft_mcq        : LLM cascade generates MCQ JSON
    Node 3 — critique         : Hallucination guard (loops up to 2x if bad)

  LLM Cascade (in priority order):
    1. Together AI → meta-llama/Llama-3-8b-chat-hf   [PRIMARY]
    2. OpenRouter  → meta-llama/llama-3-8b-instruct   [Fallback 1]
    3. Groq        → llama-3.1-8b-instant             [Fallback 2]
    4. Cohere      → command-r                         [Fallback 3]
    5. Google      → gemini-2.0-flash-lite             [Emergency]

  MCQ Output JSON format per question:
    { question, options: {a,b,c,d}, correct_option, rationale, mains_hint }

  Status: ✅ Core pipeline working (critique node is placeholder, not real LLM)

  In api/main.py, add:

    POST /api/generate-batch
    Body: { topic: str, count: int, difficulty: str }
    Action:
      1. Create a job record (job_id = UUID) in-memory or in Redis
      2. Launch asyncio background task → generate questions one by one
      3. Return immediately: { job_id, status: "pending" }

    GET /api/jobs/{job_id}
    Return: {
      status: "pending"|"generating"|"complete"|"failed",
      progress: { done: 4, total: 10 },
      questions: [ ...completed MCQs so far... ]
    }

    - Stream completed questions into the UI as they arrive (don't wait for all)
    - Mark complete when status === "complete"

    def generate_user_insights(user_id: int, db: Session) -> str:
      # Pull last 20 test sessions for this user
      sessions = db.query(TestSession).filter_by(user_id=user_id).limit(20).all()

      # Build performance summary dict
      perf = {
        "total_tests": len(sessions),
        "avg_score": mean([s.score for s in sessions]),
        "subject_scores": { compute per subject from question_results },
        "time_patterns": { group sessions by hour of day },
        "trend": [ s.score for s in sessions ]  # chronological
      }

      # Build prompt
      prompt = f"""
      You are a UPSC exam coach. Analyze this student's performance:
      {json.dumps(perf)}

      Provide a 4-section analysis in plain English covering:
      1. Weak topic clusters that need urgent revision
      2. Time-of-day performance pattern (when they perform best)
      3. Improvement velocity (which subjects are improving fastest)
      4. Specific topics to revise next (mention actual UPSC-relevant facts)

      Keep each section to 2-3 sentences. Be specific and actionable.
      """

      # Use local Ollama or cloud LLM cascade
      response = llm_cascade.invoke(prompt)
      return response.content

      - Copy connection string to .env:
          DATABASE_URL=postgresql://user:pass@host:5432/dbname

    Option 2 (Neon — serverless postgres + pgvector):
      - Create account at https://neon.tech
      - Create project → enable pgvector extension
      - Direct connection URL → paste in .env

  ── STEP 9-A.3: Prepare the project for deployment ───────────────────────────

    A) Create Procfile in project root:
         web: python -m uvicorn api.main:app --host 0.0.0.0 --port $PORT

    B) Update requirements.txt to include all production packages:
         fastapi, uvicorn[standard], sqlalchemy, psycopg2-binary,
         pgvector, langchain, langchain-community, langgraph,
         python-jose[cryptography], passlib[bcrypt], python-multipart,
         apscheduler, beautifulsoup4, httpx, celery, redis, python-dotenv

    C) Create .env.production with all secrets:
         DATABASE_URL=postgresql://...
         SECRET_KEY=your-jwt-secret-key-here-minimum-32-chars
         TOGETHER_API_KEY=...
         GROQ_API_KEY=...
         COHERE_API_KEY=...
         GOOGLE_API_KEY=...
         OPENROUTER_API_KEY=...
         REDIS_URL=redis://... (if using batch endpoint)

    D) Update api/models.py to use DATABASE_URL env variable:
         import os
         DATABASE_URL = os.getenv("DATABASE_URL", "postgresql://user:pass@localhost:5432/upsc")
         engine = create_engine(DATABASE_URL)

    E) Update CORS settings in api/main.py:
         from fastapi.middleware.cors import CORSMiddleware
         app.add_middleware(CORSMiddleware,
           allow_origins=["https://your-domain.railway.app", "http://localhost:8000"],
           allow_credentials=True,
           allow_methods=["*"],
           allow_headers=["*"]
         )

      (copy all from .env.production)

  ──────────────────────────────────────────────────────────────────────────────
  OPTION B: SELF-HOSTED VPS (Ubuntu server, more control)
  ──────────────────────────────────────────────────────────────────────────────

  Recommended providers: DigitalOcean, Hetzner, Vultr (cheapest)
  Spec needed: 2 CPU, 4GB RAM, 40GB SSD minimum

  ── STEP 9-B.1: Set up VPS ───────────────────────────────────────────────────

    1. Create Ubuntu 22.04 droplet / server
    2. SSH into server: ssh root@your-server-ip
    3. Update packages:
         apt update && apt upgrade -y
    4. Install Docker + Docker Compose:
         curl -fsSL https://get.docker.com | sh
         apt install docker-compose-plugin -y
    5. Install Nginx:
         apt install nginx -y

  ── STEP 9-B.2: Upload project ───────────────────────────────────────────────

    From your local machine:
      git init  (if not already)
      git add . && git commit -m "production ready"
      git push to GitHub

    On the server:
      git clone https://github.com/yourusername/upsc-test-series.git
      cd upsc-test-series

  ── STEP 9-B.3: Update docker-compose.yml ───────────────────────────────────

    version: '3.8'
    services:
      pgvector:
        image: pgvector/pgvector:pg16
        environment:
          POSTGRES_DB: upsc_db
          POSTGRES_USER: upsc_user
          POSTGRES_PASSWORD: ${DB_PASSWORD}
        volumes:
          - pgdata:/var/lib/postgresql/data
        restart: always

      redis:
        image: redis:alpine
        restart: always

      app:
        build: .
        ports:
          - "8000:8000"
        env_file: .env.production
        depends_on:
          - pgvector
          - redis
        restart: always

    volumes:
      pgdata:

  ── STEP 9-B.4: Create Dockerfile ────────────────────────────────────────────

    FROM python:3.11-slim
    WORKDIR /app
    COPY requirements.txt .
    RUN pip install --no-cache-dir -r requirements.txt
    COPY . .
    EXPOSE 8000
    CMD ["python", "-m", "uvicorn", "api.main:app", "--host", "0.0.0.0", "--port", "8000"]

  ── STEP 9-B.5: Configure Nginx reverse proxy ────────────────────────────────

    Create /etc/nginx/sites-available/upsc:

    server {
        listen 80;
        server_name yourdomain.com www.yourdomain.com;

        location / {
            proxy_pass http://127.0.0.1:8000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }

    Enable and restart:
      ln -s /etc/nginx/sites-available/upsc /etc/nginx/sites-enabled/
      nginx -t && systemctl restart nginx

  ── STEP 9-B.6: Add SSL certificate (HTTPS) ──────────────────────────────────

    apt install certbot python3-certbot-nginx -y
    certbot --nginx -d yourdomain.com -d www.yourdomain.com
    (Certbot auto-renews every 90 days)

  ── STEP 9-B.7: Start all services ───────────────────────────────────────────

    docker-compose up -d
    → App now live at https://yourdomain.com
